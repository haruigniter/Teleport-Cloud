Platform Overview — 2026.03 v4

Teleport-Cloud　サービス全体像

人間ユーザーとAIエージェントの両方がロボットを操作・活用できる統合プラットフォーム。Layer 2にAI基盤を追加し、段階的なAI自律化に対応する構造へ進化。

Layer 1 — Hardware
🤖 デバイス（ロボット）
現場に存在する物理デバイス。Teleport-Cloudに接続することで遠隔操作・自律動作・データ送信が可能になる。
🦾 temi　自律移動ロボット 📷 kubi　固定型デスクロボット 📱 Telepii　小型ロボット 📡 IoTセンサー類 🎥 HDカメラ（外部接続）
映像・位置情報・操作データ・センサーデータ・音声
Layer 2 — Platform Core
☁️ Teleport-Cloud
全デバイス・全アプリ・全ユーザー・全AIエージェントをつなぐ共通プラットフォーム。人間とAIの両方がロボットにアクセスする基盤。
2-A　Core Infrastructure
共通基盤
全アプリ・全ユーザー・全AIエージェントが共有するOS層。認証・デバイス管理・データ蓄積・通信基盤。
Identity & Auth Device Registry Unified Data Store Media Server（LiveKit） Event Bus Admin Dashboard
2-B　AI Infrastructure
AI実行基盤
iPresenceが作り込むAI実行環境。Guide・Field Log等のAgentが利用するLLM・プロンプト・ログ管理。
LLMランタイム（モデル管理・切替） プロンプト・コンテキスト管理 使用量・コスト管理 AIログ・トレース
1st Step：iPresence Agent強化
2-C　AI Agent Gateway
外部AIエージェント接続層
外部AIエージェント・チャットツール・顧客システムがロボットをツールとして呼び出す入口。MCP Server対応。
APIキー発行・権限管理 MCP Server（Tool公開） Teams / Slack 連携 Webhook受信
2nd Step：外部AI・チャットツール連携
3rd Step：顧客APIアクセス
2-D　Knowledge Store
知識・文脈データ層
AIが自律判断するための参照データ群。2-Bと2-Cの両方から参照される。RAG管理の実体。
施設マップ・フロア情報 業務マニュアル・FAQ ルール・制約定義 過去ログ・行動履歴（RAG）
所有モデル: 契約（組織）→ デバイス→ 管理者 / ユーザー / ゲスト / AIエージェント→ 稼働 Agent
Teleport-Cloud 上で動作するアプリケーション層（全8アプリ）

Layer 4 — Users & AI Agents
👥 ユーザー / AIエージェント（アクセス主体）
人間ユーザーは権限階層に従いアクセス。AIエージェントは2-CのAPIキー・MCP経由でアクセス。どちらも同一のデバイス・データ基盤を利用する。
iPresence（スーパー管理者）
全組織横断の監視・サポート・システム設定
組織管理者（Org Admin）
契約管理・デバイス登録・ユーザー招待・APIキー管理
ユーザー（Org User）
デバイス操作・アプリ利用・ログ参照
ゲスト（Guest）
外部専門家・他組織ユーザーの一時アクセス
AIエージェント（API / MCP）
外部AI・顧客システムがAPIキー経由でロボットをツールとして利用
Push: Webhook　/　Pull: REST API　/　MCP（外部AIエージェント）
Layer 5 — External Integration
🔌 外部システム連携
Teleport-Cloudが保有するデータを顧客の既存システムへ提供。AIエージェント経由での連携も含む。
Push — Webhook（イベント駆動）
🚨 異常検知アラート → 警備・BMS・Slack
🤖 ロボット状態変化 → 監視システム通知
💬 会話終了イベント → CRM自動登録
Pull — REST API（定期取得）
📍 位置情報履歴 → 施設管理・BIM
📋 操作・通話ログ → ERP・業務レポート
📊 分析データ → BIツール・行動解析
MCP — AI Agent連携
🧠 Claude / GPT → ロボットをToolとして呼出
💼 Teams / Slack → チャットからロボット操作
🏭 顧客AIシステム → 自律判断でロボット制御
AI自律化　3ステップロードマップ
1st Step
iPresence Agentの強化
Guide・Field Log等のAgentが2-B（AI実行基盤）と2-D（Knowledge Store）を活用し、より高度な自律判断・対話を実現する。iPresence単独で完結。
2-B AI Infrastructure + 2-D Knowledge Store
2nd Step
外部AI・チャットツール連携
Teams・Slackからの指示や外部AIエージェントが、MCP経由でロボットを自発的に操作・情報収集する。2-Dの知識を参照して自律判断。
2-C AI Agent Gateway（MCP対応）
3rd Step
顧客側からのAPIアクセス
顧客が自社のシステム・AIからTeleport-CloudのAPIを直接呼び出し、ロボットを業務フローに完全統合する。Teleport-Cloudが社会インフラ化する段階。
