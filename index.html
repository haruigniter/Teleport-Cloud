Platform Overview — 2026.03 最新版

Teleport-Cloud　サービス全体像

デバイス・プラットフォーム・アプリ・ユーザー・外部連携の5層構造。全8アプリが共通基盤の上で稼働し、データが統合されることで組織の学習基盤として機能する。

Layer 1 — Hardware
🤖 デバイス（ロボット）
現場に存在する物理デバイス。Teleport-Cloudに接続することで遠隔操作・自律動作・データ送信が可能になる。
🦾 temi　自律移動ロボット
📷 kubi　固定型デスクロボット
📱 Telepii　小型ロボット
📡 IoTセンサー類
🎥 HDカメラ（外部接続）
💻 固定タブレット・端末
映像・位置情報・操作データ・センサーデータ・音声
Layer 2 — Platform Core
☁️ Teleport-Cloud
全デバイス・全アプリ・全ユーザーをつなぐ共通基盤。Google WorkspaceにおけるGoogleアカウント＋共通データレイヤーに相当するOS層。
🔐 認証・権限管理
組織／管理者／ユーザー／ゲストの4階層管理
📋 デバイス管理
ロボットの登録・稼働状態・所属組織の管理
🗄️ データ基盤
映像・ログ・会話・位置情報の一元保存
📡 API基盤
Push（Webhook）／Pull（REST）の双方向API
🏢 組織管理
複数組織・複数拠点の横断管理
📊 管理ダッシュボード
稼働状況・ログ・レポートの一覧
LiveKit（稼働中） 通話・音声・会話ログの共通基盤 — iPresence-Agent / Telopotalk / AvatarLink で共有
所有モデル: 契約（組織）→ デバイス→ 管理者 / ユーザー / ゲスト→ 稼働 Agent
Teleport-Cloud 上で動作するアプリケーション層（全8アプリ）

Layer 4 — Users
👥 ユーザー（権限階層）
1つのアカウントで複数デバイス・複数アプリを操作。組織レベルとデバイスレベルの2階層で権限を管理。新しいアプリ追加時も再設定不要。
iPresence（スーパー管理者）
全組織横断の監視・サポート・システム設定
組織管理者（Org Admin）
契約管理・デバイス登録・ユーザー招待・請求
ユーザー（Org User）
デバイス操作・アプリ利用・ログ参照
ゲスト（Guest）
外部専門家・他組織ユーザーの一時アクセス
API連携（Push: Webhook　/　Pull: REST API）
Layer 5 — External Integration
🔌 外部システム連携
Teleport-Cloud が保有するデータを顧客の既存システムへ提供。ロボットのデータが業務基幹システムに自然に組み込まれる。
Push — Webhook（イベント駆動）
🚨 異常検知アラート → 警備・BMS・Slack
🤖 ロボット状態変化 → 稼働開始・停止・エラー通知
💬 会話終了イベント → CRM自動登録
Pull — REST API（定期取得）
📍 位置情報履歴 → 施設管理・BIM
📋 操作・通話ログ → 勤怠・業務レポート
📊 分析データ → BIツール・行動解析
